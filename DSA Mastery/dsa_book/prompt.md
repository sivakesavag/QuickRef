\#### \*\*Theory MCQ Example:\*\*

```

\*\*Q20.4: Intermediate - \[Theory] - \[Shortest Path, Dijkstra]\*\*



Why does Dijkstra's algorithm fail with negative edge weights?



A) It uses a priority queue which cannot store negative values

B) Once a node is marked visited, its distance is final — negative edges can invalidate this

C) The algorithm cannot handle cycles in graphs

D) Negative weights cause integer overflow in the distance array



\*\*Answer: B\*\*



\*\*Explanation:\*\*

Dijkstra's greedy assumption is that once a node is extracted from the min-heap with distance d, no shorter path exists. Negative edges violate this — a later-discovered path through a negative edge could yield a shorter distance to an already-finalized node. Option A is wrong (heap handles negatives). Option C is false (Dijkstra handles cycles correctly with non-negative weights). Option D is an implementation concern, not an algorithmic flaw.

```

\#### \*\*Trace MCQ Example:\*\*

```

\*\*Q17.8: Foundational - \[Trace] - \[Sorting, QuickSort, Partitioning]\*\*



Given array \[3, 1, 4, 1, 5, 9, 2, 6] and pivot = last element (6), what is the array state after one partition step (Lomuto scheme)?



A) \[3, 1, 4, 1, 5, 2, 6, 9]

B) \[3, 1, 4, 1, 5, 2, 9, 6]

C) \[1, 1, 2, 3, 4, 5, 6, 9]

D) \[2, 1, 4, 1, 5, 9, 3, 6]



\*\*Answer: A\*\*



\*\*Explanation:\*\*

Lomuto partition places all elements ≤ pivot before it. With pivot=6, elements {3,1,4,1,5,2} are all <6 and 9>6. After partition: \[3,1,4,1,5,2,6,9]. Pivot (6) lands at index 6. Note: Lomuto is less efficient than Hoare's scheme in practice (3× more swaps on average). Common confusion is between Lomuto and Hoare partition results.

```

\#### \*\*Complexity MCQ Example:\*\*

```

\*\*Q22.15: Advanced - \[Complexity] - \[DP, Knapsack]\*\*



What is the time and space complexity of the standard 0/1 Knapsack with n items and capacity W, and what optimization reduces space to O(W)?



A) O(nW) time, O(nW) space; use 1D DP array iterated in reverse

B) O(nW) time, O(W) space; use 1D DP array iterated forward

C) O(n²) time, O(W) space; use binary search optimization

D) O(nW) time, O(nW) space; cannot be reduced below O(nW) space



\*\*Answer: A\*\*



\*\*Explanation:\*\*

Standard 0/1 Knapsack is O(nW) time and O(nW) space (2D table). Space is reducible to O(W) using a 1D array, but it MUST be iterated in reverse (W to 0) to prevent using an item more than once. Forward iteration converts it into unbounded knapsack. Option B is wrong precisely because forward iteration changes problem semantics. This reversal is one of the most tested DP gotchas.

```
